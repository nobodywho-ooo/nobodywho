# nobodywho — Node.js / Electron bindings (napi-rs)

Native Node.js addon for running LLMs locally. Uses [napi-rs](https://napi.rs) to wrap the core Rust library as a `.node` addon.

## Prerequisites

- Rust toolchain (stable)
- Node.js 18+
- `@napi-rs/cli` (`npm install -g @napi-rs/cli`)
- System dependencies for building llama.cpp:
  - **Linux:** `libclang-dev cmake libshaderc-dev libvulkan-dev glslc`
  - **macOS:** Xcode command line tools (Metal support is automatic)
  - **Windows:** Vulkan SDK, Ninja, MSVC

## Project structure

```
napi/
├── Cargo.toml          # Rust crate config (napi v3, cdylib)
├── build.rs            # napi build script
├── src/
│   └── lib.rs          # Rust bindings (#[napi] annotated)
├── ts/                 # TypeScript wrapper source
│   ├── index.ts        # Re-exports native bindings + wrapper additions
│   └── streaming.ts    # streamTokens() async generator
├── dist/               # Compiled TypeScript output (gitignored)
├── tsconfig.json       # TypeScript config
├── package.json        # npm package metadata + platform targets
└── npm/                # Platform-specific package scaffolding
    ├── linux-x64-gnu/
    ├── linux-arm64-gnu/
    ├── darwin-x64/
    ├── darwin-arm64/
    └── win32-x64-msvc/
```

## Building locally

Build the native addon and TypeScript wrapper:

```bash
cd nobodywho/napi
npm install
npm run build
```

Or build steps individually:

```bash
npx napi build --release    # Build native addon
npx tsc                     # Compile TypeScript wrapper
```

This produces:
- `nobodywho.<platform>.node` — the native addon
- `index.js` — JS loader that picks the right `.node` file
- `index.d.ts` — auto-generated TypeScript type definitions
- `dist/` — compiled TypeScript wrapper (entry point for consumers)

## Building after making changes

1. Edit `src/lib.rs` (the Rust bindings)
2. Run `npx napi build --release` to rebuild the native module
3. The `.node`, `index.js`, and `index.d.ts` files are regenerated automatically
4. Run `npx tsc` to recompile the TypeScript wrapper

## Usage

```typescript
import { Model, Chat, streamTokens } from 'nobodywho';

const model = await Model.load('model.gguf', true);
const chat = new Chat(model, 'You are helpful.');

// Stream tokens with for-await
const stream = chat.ask('Hello!');
for await (const token of streamTokens(stream)) {
  process.stdout.write(token);
}

// Or get the full response
const stream2 = chat.ask('Hello!');
const response = await stream2.completed();
```

## Cross-compilation (CI)

To build for a specific target:

```bash
npx napi build --release --target x86_64-unknown-linux-gnu --cargo-flags="--locked"
```

Supported targets (defined in `package.json`):
- `x86_64-unknown-linux-gnu`
- `aarch64-unknown-linux-gnu`
- `x86_64-apple-darwin`
- `aarch64-apple-darwin`
- `x86_64-pc-windows-msvc`

## Testing

```bash
# Build first
npx napi build --release

# Run tests (no model required for sampler/utility tests)
npm test

# Run tests with a model (enables chat/streaming tests)
TEST_MODEL=/path/to/model.gguf npm test
```

## npm package layout

The main `nobodywho` package uses `optionalDependencies` to pull in the correct platform-specific
package at install time. Each platform package (`npm/<platform>/`) contains just the `.node` binary
for that platform.

## Files not committed to git

The following are build artifacts and are gitignored:
- `*.node` — compiled native addons
- `index.js` — generated JS loader
- `index.d.ts` — generated TypeScript types
- `dist/` — compiled TypeScript wrapper
- `node_modules/` and `package-lock.json`
- `npm/*/README.md` — auto-generated by napi-rs
